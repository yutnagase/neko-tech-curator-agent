# LLM 実行環境

## 概要
本プロジェクトのコアである Local LLM（ローカル大規模言語モデル） の実行環境です。  
外部のクラウドAPI（OpenAIなど）に依存せず、すべてユーザーのPC上で完結させることを最優先に設計しています。

これにより、プライバシー保護、コストゼロ、カスタマイズ性を高く保っています。

## 使用ライブラリ・技術スタック

- llama.cpp: 高性能なLLM推論エンジン（C++実装）。軽量で高速。
- langchain_community.llms.ChatLlamaCpp: LangChainとllama.cppを繋ぐラッパー。
- GGUF形式モデル: モデルファイル形式。量子化（Quantization）によりメモリ使用量を大幅削減。

## 推奨モデル

| モデル名                        | サイズ | 推奨量子化     | 対象環境               | 特徴 |
|-------------------------------|--------|----------------|------------------------|------|
| Llama-3.2-3B-Instruct     | 3B     | Q5_K_M         | 推奨（16GBメモリ〜） | バランスが良く高速 |
| Llama-3.1-8B-Instruct         | 8B     | Q4_K_M / Q5_K_M | 24GB以上               | より高品質 |
| Llama-3.1-13B-Instruct        | 13B    | Q4_K_M         | 32GB以上               | 高精度（重め） |
| Gemma-2-9B / Phi-3.5          | 9B     | Q5_K_M         | 代替候補               | 軽快 |

現時点のデフォルト推奨: `Llama-3.2-3B-Instruct-Q5_K_M.gguf`

## 設定のポイント

### 1. 温度（Temperature）
- 解説生成（Explainer Node）: `0.7` 前後  
  → 創造性と読みやすさを重視。「猫でもわかる」親しみやすい表現を引き出す。
- 自己批評（Reflector Node）: `0.4〜0.5`  
  → 客観性と厳密さを重視。
- 修正（Reviser Node）: `0.6`  
  → バランス型。

### 2. コンテキスト長（Context Length）
- 初期設定: 4096〜8192トークン
- 長いトピックやReflectionループで必要に応じて拡張（llama.cppで動的に調整可能）

### 3. その他の重要パラメータ
- Top P (Nucleus Sampling): 0.9
- Max Tokens: 解説生成時は1500〜2500程度
- Repeat Penalty: 1.1（繰り返しを抑制）

### 4. ハードウェアアクセラレーション
- NVIDIA GPU: CUDA対応（大幅高速化）
- Apple Silicon: Metal対応（Macユーザーにおすすめ）
- CPUのみ: AVX2 / AVX512対応で動作（遅めだが十分実用）

## パフォーマンス目安（Llama-3.2-3B Q5_K_M）

| 環境             | トークン生成速度 | 1トピックあたりの処理時間 |
|------------------|------------------|---------------------------|
| CPUのみ (Ryzen 7) | 25-40 t/s       | 25〜45秒                 |
| RTX 3060 / 4060   | 90-130 t/s      | 8〜15秒                  |
| M2/M3 Mac         | 70-110 t/s      | 10〜20秒                 |

## 運用Tips

- メモリが厳しい場合: 3Bモデル + Q4_K_M でさらに軽量化
- 品質を上げたい場合: 8B以上へスケールアップ + より長いコンテキスト
- モデル切り替え: `src/core/llm.py` で簡単に変更可能
- 将来的対応予定: vLLMやOllamaへの移行オプション、モデル自動ダウンロード機能

## 注意事項

- GGUFモデルは `models/` ディレクトリに配置
- 初回起動時はモデル読み込みに時間がかかる場合があります
- セキュリティのため、絶対にクラウドLLMと混在させない運用を推奨
