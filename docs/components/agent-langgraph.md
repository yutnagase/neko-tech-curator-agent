# Agent / LangGraph 詳細設計

## 1. 概要
本プロジェクトの最も重要な中核部分です。  
単なる「LLMに質問する」仕組みではなく、複数の役割を持ったAI（エージェント）が協力し合い、自己改善しながら技術解説を生成する本格的なAgentシステムを構築しています。

## 2. エージェント（Agent）とは？

エージェントとは、AIが「自律的に考えて行動する」仕組みです。

従来のChatGPTのような「1回の質問→1回の回答」ではなく、以下のような特徴を持ちます

- 目標を与えられる（例：「猫でもわかる技術解説を生成せよ」）
- ツールを使って情報を集める
- 複数ステップで考える
- 自分の出力の品質を自分で評価して改善する

本プロジェクトでは、このエージェントの考え方を LangGraph フレームワークを使って実装しました。

## 3. LangGraphとは？

LangGraphは、LangChainの拡張ライブラリで、「状態を持つ有向グラフ（Directed Graph）」として複雑なAIワークフローを定義できます。
グラフ(Graph)の詳細については、「別記：状態を持つ有向グラフ（Stateful Directed Graph）」を参照ください

ここでは、複数のAIエージェントを繋げて、分岐・ループ・状態の保持を簡単に実現できるライフラリだと大雑把に理解していただきたいです

### LangGraphを使う理由

| 方式               | 柔軟性 | ループ対応 | 状態管理 | 複雑な分岐 | 本プロジェクトでの適合度 |
|--------------------|--------|------------|----------|------------|--------------------------|
| 普通のChain        | 低     | 弱い       | 弱い     | 弱い       | 不十分                   |
| LangGraph      | 非常に高い | 非常に強い | 非常に強い | 非常に強い | 最適                 |
| CrewAI / AutoGen   | 高い   | 強い       | 中程度   | 中程度     | やや重い                 |

LangGraphの最大の強みは 「状態（State）」を一元管理しながら、条件分岐やループを自然に表現できる点です。

## 4. 本プロジェクトでの全体アーキテクチャ

```
fetch (Hacker Newsから記事取得)
  ↓
supervisor (どのエージェントを次に動かすか判断)
  ↓
explain (「猫でもわかる」解説を生成)
  ↓
reflect (AIが自分の解説を批評)
  ↓
reviser or saver (質が低い→修正、高い→保存)
  ↓
recommender (ユーザー推奨を生成)
```
※recommender機能は今後の実装にて


この流れは Stateful Graph として定義されています。

## 5. AgentState（共有状態）

すべてのノードで共有されるデータ構造です（`src/schemas/state.py`）。

```python
class AgentState(TypedDict):
    raw_topics: list[TopicMetadata]      # 取得した生トピック
    current_topic: TopicMetadata | None
    explanations: list[Explanation]      # 生成した解説
    critiques: list[Critique]            # 自己批評結果
    next: str                            # Supervisorが次に指示するノード
    revision_count: int                  # 修正回数
    date: str
```

## 6. 主要ノード詳細（具体例付き）

### Supervisor Node（監督者）
- 役割: 「次にどのノードを実行するか」を動的に決定
- 最近 `create_react_agent` + ToolNode にリファクタ中
- 例: 「このトピックはまだ解説されていない → Explainerへ」

### Explainer Node（解説生成担当）
- 役割: 「猫でもわかる」レベルで技術解説を生成
- 具体例（量子コンピュータの場合）:
  - 入力: 「Quantum Computingとは？」
  - 出力: 「猫でもわかる量子コンピュータ：猫が同時に2つの箱に入ってるような不思議な技術です...」

### Reflector Node（自己批評担当）
- 役割: 生成された解説を猫の視点で厳しく批評し、1〜100点でスコアリング
- 具体例:
  - 批評内容: 「『重ね合わせ』という言葉が難しすぎる。猫でもわかる例えが足りない。スコア: 62点」

### Reviser Node（修正担当）
- 役割: Reflectorの批評を受けて解説を改善
- Reflection Loopの肝となるノード

### Saver Node
- 役割: 最終的にSQLite + LanceDBへ保存

## 7. Reflection Loop（自己改善ループ） — 本プロジェクトの最大の特徴

単発生成ではなく、何度も改善させる仕組みです。

### 具体的な流れ（量子コンピュータの例）

1. Explainer: 初回解説を生成（60点相当）
2. Reflector: 「量子もつれの説明が抽象的すぎる。スコア68点」と指摘
3. Reviser: 指摘を受けて「猫が2匹同時に遊んでるようなイメージ」など具体例を追加して修正
4. Reflector: 再度評価 → 「スコア85点。合格！」
5. Saver: 保存

ループ条件:
- スコア70点以上 → 保存
- スコア70点未満 かつ 修正回数3回未満 → Reviserへ戻る
- 最大修正回数到達 → 強制保存（品質は担保）

この仕組みにより、出力のばらつきが大幅に減少し、品質が安定します。

## 8. 設計思想と技術的優位性

- Single Responsibility: 各ノードは1つの役割に特化
- Stateful: 処理の途中経過をすべて保持（デバッグ・再現性に強い）
- Reflection: 人間の「推敲」に近い自己改善プロセスを自動化
- Local First: すべてPC内で完結

このアーキテクチャは、2025〜2026年現在の Local Agent開発の先進事例として十分に技術アピール可能です。

## 別記：状態を持つ有向グラフ（Stateful Directed Graph）

LangGraphの核心的な概念である「状態を持つ有向グラフ」について、初心者向けにわかりやすく解説します。

### 基本的な意味

- グラフ（Graph）：点（ノード）と線（エッジ）で構成されたネットワーク構造
- 有向グラフ（Directed Graph）：線に方向（矢印）があり、処理の流れが一方向に定まっている
- 状態を持つ（Stateful）：グラフ全体で共通の状態（データ）を保持し続け、各ノードがその状態を読み書きできる

つまり、「状態を共有しながら、矢印に従って処理を進める仕組み」です。

### 本プロジェクトでの具体例

本システムでは、以下のような状態（`AgentState`）を全ノードで共有しています：

```python
AgentState = {
    "raw_topics": [...],           # Hacker Newsから取得したトピック一覧
    "current_topic": {...},        # 現在処理中のトピック
    "explanations": [...],         # これまでに生成した解説
    "critiques": [...],            # Reflectorによる自己批評結果
    "next": "explainer",           # Supervisorが次に指示するノード
    "revision_count": 2            # 修正を何回行ったか
}
```

### 処理のイメージ

1. Supervisorノードが状態の`"next"`を `"explainer"` に設定
2. Explainerノードが解説を生成 → `explanations` に追加
3. Reflectorノードが批評 → `critiques` に追加し、スコアを計算
4. 条件分岐（Conditional Edge）：
   - スコアが低い → `"next"` を `"reviser"` に変更（ループ）
   - スコアが十分 → `"next"` を `"saver"` に変更

このように、状態を更新しながらグラフを移動するのが「状態を持つ有向グラフ」です。

### なぜこの方式が優れているか

- 複雑な分岐やループ（Reflection Loop）を自然に表現できる
- 処理の途中経過がすべて状態に残るため、デバッグや再現性が非常に高い
- 各ノードを独立して開発・テストしやすい
- 将来的に新しいノードや機能を追加しやすい拡張性を持つ

従来の直線的なChain（LangChainの基本機能）では実現しにくい「条件分岐・ループ・状態の蓄積」を、グラフという論理的構造で管理できる点がLangGraphの最大の強みです。
