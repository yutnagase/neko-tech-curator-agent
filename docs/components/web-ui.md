# Web UI (Frontend)

## 概要
このプロジェクトのフロントエンドは、軽量さとメンテナンス性を最優先にした設計です。  
重厚なJavaScriptフレームワークを避け、サーバーサイドレンダリング + HTMX の組み合わせで、現代的なUI体験を実現しています。


## 技術スタック

| 技術            | 役割                              | 選定理由 |
|----------------|-----------------------------------|----------|
| HTMX       | 動的インタラクション（非同期更新） | 最小限のJSでSPAライクな体験を実現 |
| Tailwind CSS | スタイリング・レスポンシブ設計   | ユーティリティファーストで高速開発 |
| Jinja2     | テンプレートエンジン              | FastAPIと相性が良く、サーバーサイドレンダリングに最適 |
| Alpine.js  | （将来的にオプション）            | 必要最小限のクライアントサイドロジック |

## 設計思想

- 「シンプル is Best」: React / Vue / Next.js などの重いフレームワークを意図的に排除
- サーバーサイドファースト: 初期ロード速度とSEOを重視
- 猫でもわかりやすいUI: 親しみやすいデザイン（猫モチーフ、カラフルで柔らかい配色）
- Progressive Enhancement: HTMXが動作しない環境でも最低限の情報は表示可能

## 主な画面構成

### 1. トップページ (`/`)
- Heroセクション: 「猫でもわかる技術解説」のキャッチコピー + 猫イラスト
- 解説カード一覧: 最新20件程度をカード形式で表示
  - カード内容: 元タイトル、生成された解説の冒頭100文字程度、日付、いいね数
- サイドバー: カテゴリ別フィルタや「人気の解説」予定
- いいねボタン: HTMXで非同期更新（将来的にフィードバック学習へ活用）

### 2. 将来追加予定画面
- 記事詳細ページ (`/explanations/{id}`)
- 検索ページ（キーワード + ベクトル検索連携）
- おすすめページ（Recommender Agent連携）
- Aboutページ（このエージェントの説明）

## テンプレート構造
```
src/api/templates/
├── base.html          # 共通レイアウト（ヘッダー・フッター・Tailwind読み込み）
├── index.html         # トップページ
├── components/        # 将来的にカードなどの再利用コンポーネント
└── partials/          # HTMXで読み込む部分テンプレート（例: card.html）
```


## 特徴的な実装ポイント

- HTMX属性活用例:
  ```html
  <button hx-post="/like/42" 
          hx-swap="outerHTML"
          hx-target="#like-count-42">
    ❤️ いいね
  </button>
  ```

- Tailwindクラスで猫らしい柔らかいデザイン（shadow、rounded-xl、暖色系アクセント）

- レスポンシブ: モバイル対応（カードが縦に並ぶ）
